3. Описание файлов

📁 backend/database.py

Модуль работы с базой данных. 7 таблиц, 30+ функций.

Ключевые функции: create_db(), validate_fio(), validate_phone(), create_order(), cancel_order(), reserve_stock(), add_product_photo(), get_all_orders_paginated(), get_statistics_csv(), generate_invoice_html(), update_product()

📁 backend/bot.py

Основной файл бота. 8 FSM-состояний, 14 секций обработчиков.

FSM-состояния: ClientReg, StaffLogin, CreateStaff, EditStaff, NewCategory, NewProduct, AddPhotos, EditProduct, Broadcast

📁 backend/keyboards.py

Клавиатуры для 5 ролей. 7 функций генерации клавиатур.

Роли: admin, manager, consultant, client, guest

📁 backend/webapp_server.py

REST API сервер. 6 эндпоинтов + раздача статики.

API: /api/products, /api/categories, /api/my-orders, /api/photo/{file_id}, /api/health

📁 src/App.tsx

React Mini App. 6 компонентов, поиск, сортировка, карусель фото.

Компоненты: App, Header, ProductCard, PhotoCarousel, CheckoutForm, MyOrders

5. Сценарии

🛒 Сценарий 1: Клиент (быстрая регистрация + заказ)
  1. /start → приветствие
  2. 📱 Быстрая регистрация → данные из Telegram → ✅ за 1 сек!
  3. 🛍 Открыть каталог → Mini App
  4. 🔍 Ввести «диван» → демонстрация поиска
  5. 📊 Нажать «Цена ↑» → демонстрация сортировки
  6. ✅ Нажать «В наличии» → демонстрация фильтра
  7. Добавить товары в корзину → 📷 открыть карточку с каруселью фото
  8. Оформить заказ → 💳 Оплатить → ✅
  9. 📦 Мои заказы → видим заказ → ❌ Отменить (товары вернутся на склад)
  10. /start → бот узнаёт без повторной регистрации!
👔 Сценарий 2: Менеджер
  1. /staff → Менеджер → manager / manager123
  2. 📋 Заказы → фильтр «Оплаченные» → видим заказ клиента
  3. Перевод статуса: Оплачено → В обработке → Готов → Отгружен
  4. 📄 Сформировать накладную → HTML-файл ТОРГ-12 с печатью
  5. 📄 Демонстрация пагинации (если заказов > 10)
🎧 Сценарий 3: Консультант
  1. /staff → Консультант → consultant / consult123
  2. 📋 Заказы (просмотр) → видим заказы (без изменения!)
  3. 🏭 Склад → каталог и остатки
  4. 💬 Помощь клиенту → информация (доставка, оплата, возврат)
👑 Сценарий 4: Администратор
  1. /staff → Админ → admin / admin123
  2. 👥 Персонал → создать/уволить сотрудника
  3. 🏭 Склад → открыть товар → ✏️ Редактировать (цена, остаток)
  4. 📷 Добавить фото → отправить 3 фотографии
  5. 🗑 Управление фото → удалить одно фото
  6. Mini App → увидеть карусель фотографий!
  7. 📊 Статистика → 📥 Экспорт в CSV → файл для Excel
  8. 📢 Рассылка → отправить сообщение всем

6. Ключевые термины

ТерминОбъяснение
Mini App (WebApp)Встроенное веб-приложение внутри Telegram (HTML/JS/React)
FSM (конечный автомат)Пошаговые диалоги: каждый шаг = состояние, переход по действию
REST APIАрхитектурный стиль: клиент делает GET/POST запросы к серверу
CRUDCreate, Read, Update, Delete — базовые операции с данными
Валидация (Regex)Проверка данных регулярными выражениями (ФИО, телефон)
ThrottlingОграничение частоты действий пользователя (анти-спам)
ПагинацияРазбиение длинного списка на страницы (по 10 записей)
Миграция БДОбновление структуры базы данных без потери данных
ПроксированиеПересылка запроса через промежуточный сервер (фото через API)
ТОРГ-12Товарная накладная — стандартный документ для отгрузки товара
Haptic FeedbackТактильная обратная связь (вибрация) при действиях в Mini App
